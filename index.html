What is MVC?

MVC is short for Model, View, Controller, an architectural paradigm used in many languages to organize files for a website's backend. Look below for some specifics!

Explain that diagram over there, what's the flow of a request using the MVC structure?

First a client makes a request thru the browser. The browser sends that request to the router for route processing, the CRUD methods(Get, Post, Put, Delete) we all love, and the router decides what Contoller to send that request to. Once that controller or server hears that request, if needed it will access the database or Model to change, remove, or add information in the database. The database then responds back to the controller that all is well, or if there's an error. The controller will then talk to the view to render that response and send it back to the client in the browser.

So, what is the Model in MVC?

Model refers to any code pertaining to the database portion of a websites' request. Any changes, updates, general requests, all that code is placed in the Model file. The model, or database, only speaks to the controller.

So, what is the View in MVC?

View refers to anything that the client will see, basic HTML, CSS, and Javascript, as well as images. This file will talk only to the controller and render thru it, or it will get information from the controller and then render it directly to the client, depending on the language.

So, what is the Controller in MVC?

Controller refers to the middleman of the files, it connects the client/router to the database/model and to the view. Through the controller is the only way for the client and view to access the Model or database.
